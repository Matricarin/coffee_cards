# coffee_cards

## Идея

Приложение-сервис для предоставления скидочных карточек для покупки кофе в кофейнях.

## Сильные стороны проекта

- экологичность
- сокращение расходов на печать карточек
- кастомизируемый дизайн карточек
- доступ к скидочным системам разных кофеен
- оповещение пользователей о скидках

## Слабые стороны проекта 

- как убедить кофейни использовать приложение
- как убедить пользователей использователей использовать приложение

## Принцип работы

### Пользователи

Есть два типа пользователей сервисом:
- клиенты
- кофейни

Тип пользователей клиенты:
- они совершают покупки в кофейнях
- у них содержаться карточки кофеен

Тип пользователей кофейни:
- у кофеен есть филиалы
- в кофейнях работают сотрудники
- кофейни предоставляют скидочные карточки 
- кофейни продают кофе клиентам

### Сущности

Типы сущностей, которые участвуют в процессе работы приложения:

**Клиент**
- у него есть кошелек с карточками

**Карточка**
- предоставляет скидку на кофе в определенной кофейне
- имеет собственный дизайн обновляемый сотрудником

**Кошелек**
- хранилище карточек пользователя

**Сотрудник**
- продает кофе пользователю
- фиксирует факт оплаты и регистрирует скидку
- сотрудники имеют разные права доступа к корректировке и выставлению карточек

> Необходима ли сущность сотрудника, если платеж и фиксация будет производиться автоматически?

**Кофейня**
- выставляет доступные карточки на витрину
- содержит витрину с карточками

**Филиал**
- филиал также содержит витрину с доступными карточками из кофейни
- добавляет в витрину собственные карточки

### Действия пользователей

#### Действия клиентов

- клиент регистрируется в системе
- выбирает нужную кофейню
- добавляет подходящую карточку в кошелек
- совершает покупку в кофейне
- факт покупки отражается в состоянии карточки
- при выполнении условий карточки кофейня предоставляет скидку при выполнении покупки

#### Действия кофеен

- кофейни зарегистрированы в системе
- кофейням предоставляется доступ к своим личным кабинетам
- сотрудник с необходимым правом доступа выставляет нужные карточки на витрину приложения
- при добавлении карточки кофейни пользователем, кофейня получает доступ к пользователю из системы

## Привязка элементов системы к сущностям

- клиенты пользуются мобильным приложением
- кофейни работают через веб версию

- Как хранить персональные данные пользователей?
